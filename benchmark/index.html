Methodology

What "indexed" means here

Every series is shown as an index number with a base value of 100 — the same technique behind the Consumer Price Index, GDP deflators, and house-price indices. Each city's raw values are divided by its own value at a chosen base date and multiplied by 100. This shows relative movement over time, not absolute levels — which matters here because absolute levels aren't directly comparable across countries (different definitions, different geographies — see the data notes above).

Choosing the base period

Official indices fix a base period to one calendar date that never changes (e.g. CPI 2002=100). This dashboard can't do that, because each city's data starts at a different date — Ottawa's labour-force series begins in 2011, Austin's FRED series goes back to 1990. Instead, the base period is chosen per comparison, as the first date where both selected cities have data. History before that shared start is excluded from the chart — plotting Austin's 1990s data next to nonexistent Ottawa data would misinform, not inform. One consequence: Ottawa's index values are not identical across different peer comparisons, because the base date shifts depending on which peer's data availability constrains the overlap. That's a deliberate tradeoff, not an inconsistency.

Divergence Delta

This chart is the arithmetic difference between the two rebased series — Ottawa's index value minus the peer's, at each date — an index-point spread, the same technique used for relative-performance and yield-spread charts. A positive bar means Ottawa's indicator has grown faster in relative terms since the base date; it says nothing about which city has the higher absolute level.

Aligning mismatched frequencies

Sources don't all publish at the same cadence — Ottawa's labour data is monthly, Adelaide's CPI is quarterly, Ottawa's tech-sector share is annual. Where the peer series is sparser than Ottawa's, the Divergence Delta uses Ottawa's most recent index value at or before the peer's observation date — last observation carried forward, a standard technique for aligning mismatched-frequency series. This can understate short-term divergence but avoids fabricating data Ottawa doesn't have yet.

What indexing does not fix

Rebasing to 100 only removes differences in absolute level and units — it cannot correct for differences in what's actually being measured. Canadian LFS unemployment and US CPS unemployment are defined differently; Ottawa's NAICS-54 tech-sector share and Austin's CES-based professional/scientific/technical share aren't an identical concept; some "peer" series are honest substitutes rather than true equivalents (e.g. Austin's shelter_cpi is a house-price index, not a cost-of-living index, because BLS doesn't publish city-level CPI for Austin). Read the data notes above for the selected indicator before drawing conclusions from magnitude — indexing fixes comparability of scale, not of definition.

Why not year-over-year % change?

Plotting year-over-year percent change instead would avoid the base-period-choice problem entirely, and is an equally standard technique. It's not used here because it obscures the cumulative divergence since a common starting point — the more useful question for benchmarking two cities against each other over a decade-plus horizon. Both are legitimate; this dashboard optimizes for the cumulative-divergence view.
